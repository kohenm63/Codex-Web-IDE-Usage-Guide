This interactive guide will walk you through the key steps of using the Codex web-based coding environment.

Step 1: Connecting Your Universe

Codex operates on your codebase by connecting to your GitHub account. This is the first cosmic link you'll forge.

Action:

Log in to Codex and click the "Connect to GitHub" button. You'll be asked to authorize access to your repositories.

Step 2: Calibrating the Environment

An "environment" is a self-contained workspace for a single repository. It's where the magic happens.

Action:

Use the dropdown menus to select the specific GitHub repository and branch you want to work on. Then, click the "Create environment" button.

Imagine a screenshot here: A page with two dropdown menus labeled "Organization" and "Repository," and a final "Create environment" button at the bottom.

Step 3: Transmitting Instructions

The prompt box is your primary interface. Here, you communicate your coding intent in plain language.

Action:

Choose **Ask** for explanations and **Code** to generate or modify code.

Step 5: Overcoming Cosmic Glitches

Encountering an issue? Here are some common problems and their solutions to help you navigate your journey.

  • Issue: GitHub connection fails.

    Solution: Ensure you have granted the correct repository permissions on GitHub. Try revoking and re-granting access to reset the connection.

    For more details, watch the GitHub Authorization Issues video.

  • Issue: Environment setup fails.

    Solution: Codex runs in a sandboxed environment. This can sometimes fail with complex or non-standard dependencies. Try simplifying your setup or provide a custom setup script in your repository if needed.

    Learn more from this Environment Setup Troubleshooting video.

  • Issue: Codex-generated code is incorrect or doesn't work.

    Solution: This is not an oracle. Review the logs and diffs to understand the agent's thought process. Refine your prompt by being more specific, providing more context, or breaking down the task into smaller steps.

    See how to debug a real codebase in this AI Debugging Demonstration video.

  • Issue: "Failed to create task" error.

    Solution: This is often a temporary issue with the server. If it persists, try deleting and recreating the environment, or ensure your repository is not empty by adding a `README.md` file.

    For more on this, check out this Build Hour video.

  • Issue: "Binary files are not supported" error during PR creation.

    Solution: Codex may be trying to commit a binary file (like a `.pyc` or `.sqlite` file). You will need to manually remove this file from the PR and add it to your `.gitignore` file to prevent future issues.

    Find a workaround for this error in this troubleshooting video.
